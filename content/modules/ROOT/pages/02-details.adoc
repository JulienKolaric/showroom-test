== Troubleshooting Common Issues

* **OpenShift AI tab is blank**::
** Confirm the `rhods-dashboard` route exists and the user has access to the `redhat-ods-applications` namespace.

* **Inference endpoint not ready**::
** Check the InferenceService or ServingRuntime status in the OpenShift AI dashboard. Warm-up can take a few minutes on first deploy.

* **Model pull is slow**::
** Mention that production environments often use cached model storage or private registries. For the demo, narrate progress rather than dead air.

* **vLLM pod not Ready in `inference-jko`**::
** First boot downloads model weights into PVC `model-cache`. Check `oc logs deployment/tinyllama-vllm -n inference-jko`. Readiness allows up to ~10 minutes (`initialDelaySeconds: 120`, `failureThreshold: 40`).

* **Chat UI returns error**::
** Confirm `tinyllama-vllm` Service endpoints are ready before opening `vllm-chat-ui`. The nginx proxy targets `tinyllama-vllm.inference-jko.svc.cluster.local:8000`.

* **HF token secret missing**::
** Lab manifests expect Secret `hf-secret` with key `HF_TOKEN` for gated model download. Create it in `inference-jko` before deploy if required.
